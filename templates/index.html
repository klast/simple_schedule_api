simple api faq page

Параметры для любых запросов:

  • building - Здание

  • floor - Этаж

  • audience - Аудитория

  • date - Дата в формате dd.mm.yyyy

Возможные запросы:

  • Вернуть страницу, как на расписании УГАТУ, но без css стилей, с выбранным зданием, этажом, аудиторией и датой - '/schedule'

    Пример https://kurstestspelevova.herokuapp.com/schedule?&building=1&floor=4&audience=1-409&date=17.12.2019

  • Вернуть страницу с выбранным зданием, этажом, аудиторией и датой, распарсенную, и выгруженную через pandas в html - '/parse_to_table'

    Пример https://kurstestspelevova.herokuapp.com/parse_to_table?&building=1&floor=4&audience=1-409&date=17.12.2019

  • Вернуть Excel с выбранным зданием, этажом, аудиторией и датой, выгруженную через pandas в excel - '/parse_to_excel'

    Пример https://kurstestspelevova.herokuapp.com/parse_to_excel?&building=1&floor=4&audience=1-409&date=17.12.2019

    Возвращает всегда файл test.xlsx

  • Вернуть распарсенный json с выбранным зданием, этажом, аудиторией и датой, выгруженную через pandas в json - '/parse_to_json'

    Пример https://kurstestspelevova.herokuapp.com/parse_to_json?&building=1&floor=4&audience=1-409&date=17.12.2019

    Может выглядеть не корректно в браузерах на первый взгляд, но это
    Самый распространенный формат общения клиента с сервером

Возможные ошибки при работе:

  • Приложение может вернуть ошибку "* arg dont passed"
    это значит забыли прописать какой-то из аргументов

  • Приложение может вертуть ошибку "for row in test.find_all('tr'): AttributeError: 'NoneType' object has no attribute 'find_all'"
    Это может значит либо то, что аргументы введены не правильно, и расписание выдано пустое, либо просто почему-то не выдалась страница, и надо попробовать ещё раз.
